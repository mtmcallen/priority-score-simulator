Ambient Listening Consent by Roundable Type — Design Spec

Extend Ambient Listening consent configuration and mobile consent modals to support patient, location, and staff rounding — with separate suggested scripts per roundable type and an optional No Consent Needed action per type.

Evolve · Provider Settings CipherRounds Mobile Ambient Listening Patient · Location · Staff Interactive prototype ↗

Summary

Today, care providers configure a single Ambient Listening Consent Suggested Script in Provider Settings. Mobile shows a patient-specific consent modal with hardcoded default script lines and the instruction “Please ask the patient for verbal consent before starting the recording.”

This spec splits consent configuration by roundable type (patient, location, staff), updates the mobile consent modal copy per type, and adds an optional third footer button — No Consent Needed — that organizations can enable independently for each type.

  • Evolve: Three type-specific consent script fields, each with its own No Consent Needed Button Enabled checkbox directly below (patient, location, staff).
  • Mobile: Show the instruction and suggested script that match the roundable type; stop using hardcoded script lines.
  • Previously declined: Show the existing opt-out banner for patient, staff, and location roundables when applicable.

Multiline behavior: Each script field accepts free text. Mobile splits on line breaks into separate suggested-script lines (same behavior as today’s single field). Empty lines are ignored.

Location rounding context

Location rounds are often not a 1:1 conversation with the location itself. Common patterns include staff group audits, compliance walk-throughs, and group check-ins. When staff are present, use the suggested script and obtain consent. Organizations often enable Location Ambient Listening No Consent Needed Button Enabled for empty walk-through audits while leaving patient/staff checkboxes off.

Previously declined banner

When the roundable previously declined recording, show the existing banner above the instruction for patient, staff, and location roundables:

  • Title: {Roundable name} previously declined recording
  • Body: Opted out of audio recording on their last round.

“No Consent Needed” button behavior

Controlled per roundable type by the checkbox under each consent script field in Evolve. When enabled for a type, that type’s consent modals include a third footer button. Instruction text never tells the rounder to select it.

  • Does not mark the roundable as having declined recording.
  • Allows recording to start without capturing verbal consent.
  • Used for workflows where consent is not applicable (e.g., empty location walk-through audits).

Layout (when enabled): Three equal-width footer buttons — Consent Given · Declined · No Consent Needed. When disabled, two equal-width buttons — Consent Given · Declined. At large accessibility text sizes, buttons stack vertically.

Large accessibility text

When the rounder uses maximum iOS Dynamic Type (or equivalent), consent modal text scales up and footer buttons stack full-width. See the interactive prototype to review this layout.

Acceptance Criteria

  1. Evolve shows three script fields; each has a No Consent Needed Button checkbox directly below it.
  2. Each roundable type independently shows 2 or 3 footer buttons based on its Evolve checkbox.
  3. Instruction copy never references the No Consent Needed button.
  4. Patient, location, and staff modals show type-appropriate instruction and suggested script.
  5. Consent Given dismisses the modal and allows recording to proceed.
  6. Declined dismisses the modal, blocks recording, and marks the roundable as declined for future rounds.
  7. No Consent Needed (when enabled) dismisses the modal and allows recording without marking the roundable as declined.
  8. Previously declined roundables show the declined banner on the next consent attempt.
  9. Mobile no longer uses hardcoded suggested script lines when a configured script is available.
  10. If no suggested script is configured, hide the script block; still show instruction and action buttons.
