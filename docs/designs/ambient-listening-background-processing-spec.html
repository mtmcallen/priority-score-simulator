Ambient Listening Background Processing — Design Spec

Allow rounders to dismiss the processing modal and continue using CipherRounds while an Ambient Listening recording uploads and processes. Recording bar, patient list status, and toast notifications reflect progress and outcomes.

CipherRounds Mobile Ambient Listening Background processing

Summary

Today, when a rounder taps Stop & Submit, a full-screen Processing modal appears and the interaction screen stays blocked until upload and server-side processing finish (or fail).

This enhancement adds an opt-in path: the rounder can check Finish processing in the background on the processing modal, dismiss the blocker, and move freely in the app (or leave the app) while processing continues. The recording bar and patient worklist communicate state; toasts confirm success or failure.

  • Blocking path unchanged — if the rounder does not check the box, behavior matches today (modal stays until complete).
  • Background path — modal dismisses; bar shows Processing in Progress; user can navigate anywhere.
  • On success — bar returns to Recording Complete with Start (append) and clear (×); worklist shows Needs Review; green toast.
  • On failure — bar shows Processing Failed with Retry; worklist still Needs Review; red toast.

Patient worklist — Needs Review

When processing completes—success or failure—the patient’s worklist card shows Needs Review as the last-round time/status (existing AL pattern).

  • Update local interaction state when server returns finished (success) or failed terminal state.
  • Refresh worklist card even if user is on a different tab or screen.
  • Success: rounder can open interaction to review AI-drafted answers.
  • Failure: rounder still sees Needs Review but bar on interaction shows Retry path.

Acceptance criteria

  1. After Stop & Submit, processing modal appears with checkbox Finish processing in the background.
  2. Without checking the box, screen remains blocked until processing finishes (regression).
  3. With checkbox selected, modal dismisses and bar reads Processing in Progress; user can navigate app-wide.
  4. On success: green toast, bar Recording Complete with Start, worklist Needs Review.
  5. On failure: red toast, bar Processing Failed with Retry, worklist Needs Review.
  6. Start appends a new recording segment; × clears the current recording.
  7. Background processing continues if user backgrounds OS or switches patients.
